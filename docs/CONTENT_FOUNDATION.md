# Content foundation and transcript synthesis

## Central problem

The previous website had a clean visual identity but an unstable content model. It described a generic software engineer rather than Qiaomai’s actual thermal-management work, reduced Core to engineering, reduced Growth to AI/data, and presented invented experiment logs as historical records.

The rebuilt site uses one central sentence:

> You are not a job or a single skill. You are a set of human-capital assets that can depreciate, appreciate, drift, and be rebalanced.

## Stable framework after synthesis

| Position | Function | Central question | Asset evidence |
|---|---|---|---|
| Core | Protect the principal and keep the system operational | What keeps me standing? | Sustainable work, recovery, responsibility, and real room to choose |
| Growth | Build future optionality | What lets me keep growing? | New deployable capability, projects, roles, or paths |
| Distribution | Bring internal value into the outside world | How does my value become visible? | Artifacts that can be found, shared, cited, trusted, or used |
| Meta | Manage and rebalance the portfolio | How do I know I am not drifting? | Visible trade-offs and evidence-based allocation changes |

The system loop is:

**Protect → Grow → Distribute → Rebalance**

## Editorial decisions

The rebuilt website intentionally removes:

- fixed allocation percentages presented as general advice;
- unsupported human-capital return rates and speculative compound-growth charts;
- fictional monthly notes and fabricated Q1 allocation audits;
- software-engineering and code-review claims that conflict with the author’s actual work;
- placeholder newsletter, email, YouTube, X, and experiment URLs;
- anonymous “engineer” positioning that hides the author.

The website preserves:

- the ETF portfolio analogy;
- the worker’s real dependence on wages and current cash flow;
- the four functional positions;
- health as the true principal beneath Core;
- project/output conversion as the test for Growth and Distribution;
- rebalancing as the primary Meta operation;
- the author’s engineering perspective and first-person intellectual history.

## Ongoing update rule

When a new transcript introduces a new idea:

1. preserve the concrete scene and source;
2. identify the question it actually answers;
3. decide whether it changes a definition, an example, or only the author’s current allocation;
4. update the parent `01_FRAMEWORK` files if the definition changes;
5. update Chinese and English website copy together;
6. run typecheck, lint, build, and production dependency audit.
